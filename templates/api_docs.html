POST /api/chat
Send a message to the AI. If session_id is provided, continues existing conversation. If not, creates a new session.

Request Body:

message string The user's message to send to the AI (required)
session_id string Optional session ID to continue an existing conversation
temperature float Controls randomness in AI responses (0.0 to 1.0, default: 0.7)
max_tokens int Maximum tokens in response (-1 for unlimited, default: -1)
{ "message": "What is cotton candy?", "session_id": "abc123-def456-ghi789", "temperature": 0.7, "max_tokens": -1 }

Response:

Returns session ID, AI response message, and complete conversation history.

GET /api/sessions/{session_id}
Retrieve a complete chat session including all messages and timestamps.

Path Parameters:

session_id string The unique identifier of the session to retrieve

Response:

Returns complete session data with all messages, timestamps, and metadata.

GET /api/sessions
Get a list of all available chat session IDs, sorted by most recent first.

Response:

Returns an array of session IDs that can be used to retrieve specific sessions.

GET /api/sessions/{session_id}/messages
Retrieve only the messages from a specific session without session metadata.

Path Parameters:

session_id string The unique identifier of the session

Response:

Returns a list of messages in chronological order.

DELETE /api/sessions/{session_id}
Permanently delete a chat session and all its messages.

Path Parameters:

session_id string The unique identifier of the session to delete

Response:

Returns success message if deletion was successful.

GET /api/health
Check if the application and LM Studio are running properly.

Response:

Returns status information about the service and LM Studio connectivity.
